# 🚀 Notification System - Quick Start Guide

## For Developers

### 1️⃣ Understanding the System

**What is it?**
A comprehensive notification system that handles 11 different notification types from the backend (Django) and displays them to users on the frontend (React/Next.js).

**Where are they displayed?**
- 🔔 **Notification Bell** (top right header) - Shows 5 recent notifications
- 📬 **Notifications Page** (`/notifications`) - Shows all notifications with infinite scroll
- 🏆 **Leaderboard Modals** - Special modals for leaderboard/achievement notifications\n\n---\n\n## 2️⃣ Supported Notification Types\n\n```\n📊 LEADERBOARD  → Weekly rankings\n🏆 ACHIEVEMENT  → Top 3 winners + bonus points\n⭐ SYSTEM       → Milestone achievements\n❤️ LIKE         → Story/verse likes\n💬 COMMENT      → New comments\n✨ VERSE        → Verse contributions\n📖 STORY        → New stories from followers\n👤 FOLLOW       → New followers\n@  MENTION      → User mentions\n💾 SAVE         → Story saves\n🎉 WELCOME      → Welcome message\n```\n\n---\n\n## 3️⃣ How to Use in Your Component\n\n### Import the utilities\n```javascript\nimport {\n  isLeaderboardNotification,\n  isAchievementNotification,\n  getNotificationRoute,\n  formatNotificationDisplay,\n  getNotificationTypeConfig\n} from 'lib/leaderboardNotifications';\n```\n\n### Check notification type\n```javascript\nif (isLeaderboardNotification(notification)) {\n  console.log('This is a leaderboard update');\n}\n\nif (isAchievementNotification(notification)) {\n  console.log('This is an achievement!');\n}\n```\n\n### Get routing info\n```javascript\nconst route = getNotificationRoute(notification);\nrouter.push(route);\n// Example: /leaderboard, /stories/my-story, /username, etc.\n```\n\n### Format for display\n```javascript\nconst displayData = formatNotificationDisplay(notification);\nconsole.log(displayData.icon);     // '📊'\nconsole.log(displayData.title);    // 'Leaderboard Update'\nconsole.log(displayData.color);    // 'from-cyan-400 to-blue-500'\n```\n\n### Get type config\n```javascript\nconst config = getNotificationTypeConfig('LIKE');\nconsole.log(config.icon);              // '❤️'\nconsole.log(config.defaultGradient);   // 'from-red-400 to-pink-500'\n```\n\n---\n\n## 4️⃣ Common Tasks\n\n### Display a notification\n```javascript\nconst { icon, title, subtitle, color } = formatNotificationDisplay(notification);\n\nreturn (\n  <div className={`bg-gradient-to-r ${color} bg-opacity-10 p-4 rounded-lg`}>\n    <div className=\"text-2xl\">{icon}</div>\n    <div className=\"font-bold\">{title}</div>\n    <div className=\"text-sm text-gray-400\">{subtitle}</div>\n  </div>\n);\n```\n\n### Handle notification click\n```javascript\nconst handleClick = async (notification) => {\n  // Mark as read\n  if (!notification.is_read) {\n    await notificationsApi.markAsRead(notification.id);\n  }\n  \n  // Navigate to relevant page\n  const route = getNotificationRoute(notification);\n  router.push(route);\n};\n```\n\n### Show all notifications\n```javascript\nnotifications.map(notification => (\n  <div\n    key={notification.id}\n    onClick={() => handleClick(notification)}\n  >\n    {formatNotificationDisplay(notification).icon}\n    {notification.title}\n  </div>\n))\n```\n\n---\n\n## 5️⃣ Adding a New Notification Type\n\nIf you need to add a new notification type:\n\n### Step 1: Add type config\n```javascript\n// In lib/leaderboardNotifications.js\nconst notificationTypeConfig = {\n  // ... existing types\n  MY_NEW_TYPE: {\n    icon: '🎯',\n    defaultGradient: 'from-purple-400 to-pink-500',\n    defaultBg: 'bg-purple-500/10',\n    defaultBorder: 'border-purple-500/30'\n  }\n};\n```\n\n### Step 2: Add detection function (if needed)\n```javascript\nexport const isMyNewType = (notification) => {\n  return notification?.notification_type === 'MY_NEW_TYPE';\n};\n```\n\n### Step 3: Update routing (if needed)\n```javascript\nexport const getNotificationRoute = (notification) => {\n  // ... existing checks\n  if (notification.notification_type === 'MY_NEW_TYPE') {\n    return '/my-new-page';\n  }\n  // ... rest of function\n};\n```\n\nThat's it! Your new type will work everywhere automatically.\n\n---\n\n## 6️⃣ Data Structure\n\n### Notification Object\n```javascript\n{\n  id: 123,\n  recipient_id: 456,\n  notification_type: 'LEADERBOARD',\n  title: 'You\\'re the #1 Champion!',\n  message: 'Weekly leaderboard update...',\n  is_read: false,\n  time_ago: '2 hours ago',\n  created_at: '2025-11-25T15:30:00Z',\n  \n  // Optional fields (depends on type)\n  sender: { id: 789, username: 'alice', ... },\n  story: { id: 100, slug: 'my-story', title: 'My Story', ... },\n  verse: { id: 101, slug: 'verse-1', ... },\n  \n  // Leaderboard-specific\n  rank: 1,\n  position: 1,\n  finalized_score: 500,\n  score: 500,\n  weekly_score: 500,\n  lifetime_score: 2000\n}\n```\n\n---\n\n## 7️⃣ Testing in Development\n\n### Create a test notification\n```bash\n# Open Django shell\npython manage.py shell\n\n# Create a test notification\nfrom api.models import Notification\nfrom django.contrib.auth.models import User\n\nuser = User.objects.first()\nNotification.objects.create(\n    recipient=user,\n    notification_type='LEADERBOARD',\n    title='Test Leaderboard',\n    message='This is a test leaderboard notification',\n    rank=1,\n    finalized_score=500\n)\n```\n\n### Check it appears in frontend\n1. Verify `notification_type` matches one of the 11 types\n3. Check WebSocket connected: Look for \"WebSocket connected\" in console\n4. Refresh the page\n\n### Issue: Wrong icon showing\n**Solution:**\n1. Verify `notification_type` in database\n2. Check `getNotificationTypeConfig()` has the type\n3. Clear browser cache: `Ctrl+Shift+Delete`\n4. Check browser console for errors\n\n### Issue: Navigation not working\n**Solution:**\n1. Verify story/verse exists\n2. Check `getNotificationRoute()` returns valid path\n3. Test manual navigation: `router.push('/stories/test')`\n4. Check notification has required fields\n\n### Issue: Performance slow\n**Solution:**\n1. Check notification list isn't huge (>1000 items)\n2. Verify images loading: DevTools Network tab\n3. Check for infinite loops: DevTools Performance tab\n4. Profile Celery tasks: Are they completing?\n\n---\n\n## 9️⃣ Key Files Reference\n\n| File | Purpose | Size |\n|------|---------|------|\n| `lib/leaderboardNotifications.js` | Utility functions | 350+ lines |\n| `components/header/NotificationBell.js` | Dropdown bell | 350 lines |\n| `components/LeaderboardNotificationDisplay.js` | Special modal | 200 lines |\n| `app/notifications/NotificationsClient.js` | Full page | 500 lines |\n| `NOTIFICATION_SYSTEM_COMPLETE.md` | Full docs | 500+ lines |\n| `NOTIFICATION_QUICK_REFERENCE.md` | Quick lookup | 300+ lines |\n\n---\n\n## 🔟 Environment Variables\n\nNo new environment variables needed!
